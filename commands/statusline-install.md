Ниже — вывод движка бэклога по установке statusLine. Сырой вывод пользователю НЕ
показывается: он инжектится только в твой контекст. Пользователь увидит лишь то, что
ты сам напечатаешь в ответе.

Передай результат дословно: что именно произошло (создан новый statusLine / бэйдж
добавлен поверх существующего / уже было подключено / прервано из-за невалидного JSON),
и путь к wrapper-скрипту. Если движок сообщил про бэкап настроек — упомяни его.

Скажи пользователю: бэйдж появится в новой сессии Claude Code (statusLine
перечитывается при старте). Откат — командой `/backlog:statusline-uninstall`.

Если вывод начинается с `error:` — покажи текст ошибки и не заявляй, что всё готово.

!`python3 "${CLAUDE_PLUGIN_ROOT}/scripts/backlog.py" statusline install 2>&1 || true`
